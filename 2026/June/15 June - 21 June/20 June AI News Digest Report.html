AI Safety & Research

Google DeepMindThe first-ever AI Control Roadmap: 15 system-level defenses, defense-in-depth, ~1M internal coding-agent trajectories analysed

Published 18 June 2026 · Primary sources: DeepMind blog, GDM AI Control Roadmap PDF, Fortune, eWeek

Google DeepMind published its first AI Control Roadmap on 18 June, authored by Rohin Shah and John "Four" Flynn (DeepMind's VP of Security & Privacy), with the central proposition that model alignment cannot be assumed reliable at the capability levels DeepMind is now shipping internally, and therefore system-level containment must carry the weight that classifier-level safety has historically been asked to carry alone. The framework identifies 15 system-level defenses across two capability axes: Detection (D1–D4) and Prevention & Response (R1–R3). Concrete examples named in the post: chain-of-thought monitoring and asynchronous alerts for present-day capability, real-time response and agent access control for near-term capability, and activations monitoring and shutdown infrastructure for advanced capability. Flynn disclosed that DeepMind has already built an internal prototype that asynchronously monitors coding-agent trajectories and escalates suspicious findings for human review, and that DeepMind has analysed ~1 million internal coding-agent tasks — most flagged events involve misinterpretation or overeagerness rather than adversarial intent. The roadmap is the first time a frontier lab has published a defence-in-depth schema explicitly designed for internal agent deployment risk (versus the external-deployment Responsible-Scaling-Policy genre most labs have published to date), and it directly anticipates the new question DeepMind is preparing to answer at the next capability tier: how do you safely run agents inside your own production codebase?

OpenAI / Boston Children'sNEJM AI: OpenAI o3 Deep Research surfaces new diagnoses in 18 of 376 previously-unsolved pediatric rare-disease cases (+4.8% yield)

Published 18 June 2026 · Primary sources: OpenAI newsroom, NBC News, NEJM AI

A peer-reviewed study published in NEJM AI on 18 June — co-authored by Boston Children's Hospital, Harvard Medical School, and OpenAI — reports that OpenAI o3 Deep Research, used as an evidence-synthesis tool alongside human clinicians and re-genome-analysis, surfaced credible diagnoses in 18 of 376 children whose rare-disease cases had previously been worked through by specialists without a landing diagnosis. The added diagnostic yield is +4.8%. The 18 new diagnoses break down as 10 neurodevelopmental, 4 neuromuscular, 2 sudden-death cases retrospectively diagnosed, and 2 early-childhood psychosis. Methodologically: the model received clinician notes, patient symptoms, and a pre-filtered list of candidate genes for each case, and was asked to surface evidence-linked candidate explanations — not to make any clinical decision. Every surfaced lead was independently assessed by human specialists, with additional confirmatory testing performed where indicated. The framing OpenAI emphasises in its newsroom post is careful and worth quoting: "The model did not diagnose any patient or make any clinical decision." What it did was raise the floor on how many candidate explanations a clinical team can plausibly survey within the time budget of a single revisit of a previously-stalled case — an architecturally significant change for any condition where the bottleneck is hypothesis-generation rather than confirmatory test availability.

Developer Platforms & Coding Agents

AnthropicClaude Code v2.1.183 (19 June): auto-mode blocks destructive git / IaC commands by default, plus enterprise-managed MCP via Okta

Released 19 June 2026 · Primary source: code.claude.com/docs/en/changelog

Anthropic shipped Claude Code v2.1.183 on 19 June with a meaningful new agent-safety surface that maps directly onto the DeepMind AI Control Roadmap framing covered above. The headline change: auto mode now blocks destructive Git commands by default when the user did not explicitly ask for them — git reset --hard, git checkout -- ., git clean -fd, and git stash drop — and additionally blocks git commit --amend when the commit being amended was not made by the agent in the current session. The same release extends the same default-deny posture to infrastructure-as-code teardown: terraform destroy, pulumi destroy, and cdk destroy now require an explicit user-named stack. Secondary changes include: a new deprecation warning when the requested model has been auto-updated to a successor (visible on stderr in -p mode and now propagated to models set in agent frontmatter), an attribution.sessionUrl setting to omit the claude.ai session link from commits and PRs in web and Remote Control sessions, and a /config --help surface to list shorthand keys. Notable bug-fix: WebSearch returning empty results when invoked from subagents was tracked down and fixed. In parallel, Anthropic announced enterprise-managed MCP connector access starting with Okta — administrators provision connectors once for the whole organization through their identity provider; users get connector access on first login. The net effect for any team operating a long-running Claude Code agent inside their production developer environment is that the "agent runs git reset --hard on a mis-routed instruction" failure mode — a real production incident pattern over the past three weeks — is now closed by default.

Enterprise Security

Microsoft 365 Copilot"SearchLeak" (CVE-2026-42824): the canonical case study for indirect prompt injection meeting legacy SSRF

Disclosed 15 June, server-side patch 11 June 2026 · Enterprise-security wave through 18–19 June · Primary sources: Varonis Threat Labs, BleepingComputer, SC Media

Varonis Threat Labs publicly disclosed "SearchLeak" — tracked as CVE-2026-42824, rated critical — on 15 June, after Microsoft pushed a server-side fix to all Microsoft 365 Copilot Enterprise tenants on 11 June as part of its June Patch Tuesday cycle. The vulnerability is the now-canonical worked example of indirect prompt injection meeting legacy server-side request forgery: a three-stage chain that combines a parameter-to-prompt injection in a Copilot Enterprise Search URL, an HTML rendering race condition, and a content-security-policy bypass enabled by a long-standing Bing SSRF. The exploit shape is one-click: the victim sees only "Copilot thinking" with no visible data movement, while the chained payload exfiltrates mailbox content, calendar entries, OneDrive and SharePoint files, MFA codes, and meeting details. Microsoft's fix sits in Copilot's orchestration engine — it now sanitises and explicitly separates user prompts from any embedded instructions present in retrieved content. For enterprise security teams, the post-disclosure week has been spent walking the same threat model across other Copilot- and agent-style integrations that combine retrieval, browser rendering, and any externally-influenced parameter surface; the assumption that the LLM-on-top is the security boundary is the failure-mode SearchLeak generalises. This is the second-ever AI-feature CVE in the Microsoft 365 surface and the first to be rated maximum-severity.
