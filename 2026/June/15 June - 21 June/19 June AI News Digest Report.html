Top Story — Google Forces Gemini CLI Users onto Antigravity CLI

GoogleGemini CLI stops serving Pro, Ultra, and free Code Assist users on 18 June — Antigravity CLI is the only path forward

18 June 2026 · Primary sources: developers.googleblog.com, google-gemini/gemini-cli GitHub Discussions #27274

Google’s long-telegraphed deadline arrived on Thursday 18 June: the Gemini CLI and the Gemini Code Assist IDE extensions stopped serving requests for all Google AI Pro, Ultra, and free Code Assist for Individuals users. Enterprise customers on Gemini Code Assist Standard / Enterprise, and developers using paid API keys or Google Cloud projects, retain access — everyone else has been pushed onto the new Antigravity CLI, a Go-based rewrite announced at Google I/O on 19 May 2026. The new CLI shares the agent engine used by the Antigravity 2.0 desktop app and includes Agent Skills, Hooks, and Subagent APIs ported from the old harness. The transition is contentious for two reasons. First, the original google-gemini/gemini-cli repo was Apache-2.0 with an active outside-contributor community, and the replacement is being shipped as a closed-source binary — community comment on GitHub Discussion #27274 includes the now-recurring complaint that “we’re making the project closed source” after years of volunteer contributions. Second, reports from early Antigravity CLI users describe markedly lower free-tier quotas (a 7-day refresh cycle versus the more permissive Gemini CLI allowance), which functionally retires the “free agentic coding” tier that Gemini CLI provided. For coding-agent builders, this is the first major frontier-lab CLI deprecation rather than upgrade — and a useful signal about how the platform-distribution surface for agents will be consolidated going forward.

15–18 June 2026 · Primary sources: aws.amazon.com/whats-new, docs.aws.amazon.com, x.ai/news

xAI shipped Grok 4.3 to Amazon Bedrock on Monday 15 June, with the AWS-side blog launch on Tuesday 16 June and broad developer-tier rollout completing by Thursday 18 June. It is xAI’s first model ever to ship on Bedrock, an architecturally significant move given xAI’s recent SpaceX-led NASDAQ listing as SPCX (covered in the 13 June digest). Specs are crisp: 1,000,000-token context window, configurable reasoning effort across none / low / medium / high, native tool calling, structured outputs, and response streaming. The pitch is enterprise-grade reasoning: xAI cites the lowest hallucination rate of any frontier reasoning model on internal evals and pitches the model at contract review, case-law research, credit-agreement analysis, and long-document financial Q&A. Bedrock pricing on the on-demand tier is $1.25 per million input tokens, $2.50 per million output tokens, with cached input at $0.20/M — aggressively below GPT-5.5 and Gemini 3.5 Flash on the input side. The model runs on Bedrock’s new Mantle inference engine, which AWS positions as a price-performance-optimised path for third-party frontier models. For agent builders, the practical effect: xAI is no longer an X-platform-only model surface — Bedrock IAM, VPC peering, and Guardrails now apply to Grok in the same way they do to Claude and Llama.

Security & Software Engineering

Node.jsMulti-CVE security release across 22.x / 24.x / 26.x — TLS hostname-verification bypass, HTTP/2 ORIGIN-frame DoS, WebCrypto integer overflow

18 June 2026 · Primary sources: nodejs.org security advisory, OpenJS Foundation

The Node.js project shipped a multi-CVE security release on Thursday 18 June, with simultaneous patched builds across the 22.x, 24.x, and 26.x release lines (specifically 22.23.0, 24.17.0, and 26.3.1). The most operationally serious item is CVE-2026-48618, a TLS hostname-verification bypass caused by improper handling of Unicode dot separators in TLS hostname normalisation — a mismatch between the resolver and verifier paths that, in the worst case, lets an attacker satisfy a wildcard-cert match for a hostname the certificate was never intended to cover. Two further high-severity items: (i) an HTTP/2 ORIGIN-frame Out-of-Memory denial-of-service against Node.js HTTP/2 clients (an attacker-controlled server can stream unlimited ORIGIN frames until the client OOMs), and (ii) a WebCrypto integer overflow that crashes the process when subtle.encrypt() input length is an exact multiple of 2 GiB. The release also bundles dependency upgrades that matter independently: llhttp 9.4.2, nghttp2 1.69.0, OpenSSL 3.5.7, and per-line undici bumps. Production operators on any of the three supported lines should prioritise the upgrade this week — particularly anyone running Node.js HTTP/2 clients against untrusted upstreams.

16 June 2026 · Primary sources: TechCrunch, CNBC, NBC News, AI Business

On Monday 16 June — four days after the record-setting SPCX Nasdaq listing covered in the 13 June digest — SpaceX announced a $60 billion all-stock acquisition of Anysphere, the developer of Cursor. By transaction size it is the largest acquisition of a venture-backed startup in history. The deal does not use SpaceX’s IPO proceeds; close is targeted for Q3 2026. The strategic logic is straightforward: SpaceX’s xAI division has competed at the model layer against Anthropic and OpenAI but lacks an agentic-coding distribution surface to rival GitHub Copilot or Claude Code. Cursor brings that surface plus serious revenue — reporting ~$2.6 B in annualised B2B revenue, with enterprise sales growing sharply and a customer list anchored by Stripe, Adobe, and NVIDIA (whose CEO Jensen Huang has publicly called Cursor his “favourite enterprise AI service”). The deal also pushes the IDE-and-agent landscape into sharper consolidation: Microsoft owns GitHub Copilot, Anthropic owns Claude Code, OpenAI now owns Codex plus Ona (covered in the 13 June digest), and SpaceX/xAI now owns Cursor — leaving Google’s newly-deprecated-and-reborn Antigravity stack (see top story above) as the last frontier-lab coding-agent surface without a major M&A consolidation event on its books.
