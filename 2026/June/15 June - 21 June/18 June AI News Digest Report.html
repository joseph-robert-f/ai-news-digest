AnthropicAnthropic opens its third APAC office in Seoul as Chaoui says the Fable 5 / Mythos 5 export-control directive will lift within days

Anthropic on Wednesday 17 June formally opened its Seoul office, its third in Asia-Pacific after Tokyo and Bengaluru. The office is led by KiYoung Choi, a three-decade Korean tech-business veteran. Anchor deployments named in the announcement: NAVER rolling out Claude Code across its full engineering organization; Samsung SDS running Claude Cowork and Claude Code across Samsung Electronics; LG CNS deploying Claude across LG Group; Nexon using Claude Code for live-service game development; Hanwha Solutions via AWS Bedrock with in-region data controls; and Channel Corp embedding Claude inside the Channel Talk platform that serves 230,000+ businesses. That is the most specific public timeline anyone from Anthropic has given since Monday's Washington meetings with Commerce, the CIA, and White House science advisor Michael Kratsios. The Seoul launch is the structural hedge against the DC-political downside: if foreign-national gating becomes the steady-state condition of access to frontier Claude, Anthropic now has an in-country presence in the most concentrated cluster of large-scale enterprise customers it serves outside the US.

Developer Platforms & Coding Agents

GitHubGitHub Copilot App hits GA on macOS, Windows and Linux — canvases, Agent Merge, and cloud automations

17 June 2026 · Primary source: GitHub Changelog & The GitHub Blog · Confirmed by TechTarget, DevOps.com, WinCentral

GitHub took the Copilot App — previously in technical preview since May — to general availability on Wednesday 17 June for macOS, Windows and Linux. The app is GitHub's first standalone, agent-native desktop client; sessions launch from a GitHub issue, pull request, or free-form prompt, and can be fan-out across multiple repos in parallel, each on its own branch and git worktree. Three pieces are new since the May preview. First, canvases: bidirectional shared surfaces where agent and human operate on the same plan, PR, terminal, or browser session — the company's bet that progress should be "visible and steerable instead of buried in chat." Second, Agent Merge: monitors CI, tracks required reviewers, addresses failing checks, and waits for all merge conditions to be satisfied before completing a PR. Third, cloud automations: schedule recurring agent work in the cloud without needing your laptop awake. Bring-your-own-model is wired in: the model behind each session is configurable per-session, and external tools attach via MCP servers. Access on Copilot Business or Enterprise plans requires admins to have Copilot CLI enabled in policy settings. This lands squarely in the same surface as Cursor (now a SpaceX subsidiary — see the next item), Codex CLI, Claude Code, Cline, Replit Agent, and the freshly-renamed Antigravity CLI — but with GitHub's structural advantage that the orchestration plane is inside the same identity, repo, CI, and review graph the developer already uses. TechTarget's coverage notes early enterprise reliability and pricing concerns; both will be the next 30 days' tells.

GoogleAntigravity CLI cutover today (18 June): four silent breaking changes for anyone scripting Gemini CLI

Cutover 18 June 2026 · Primary source: Google Developers Blog · Confirmed by The Register, OSTechNix, Groundy, InventiveHQ, Virtualization Review

Today, 18 June 2026, Google's Gemini CLI stops serving requests for AI Pro, AI Ultra, and free Gemini Code Assist users. The replacement, Antigravity CLI, was first announced at Google I/O and quietly walks users from the open-source Gemini CLI to a closed-source rewrite in Go with a new binary, a new config layout, and a quota model that trades daily limits for weekly ones. Enterprise customers are exempt: organisations on Gemini Code Assist Standard or Enterprise, or using paid Gemini / Gemini Enterprise Agent Platform API keys, retain uninterrupted access to Gemini CLI for an unspecified runway. For everyone else, four breaking changes will silently catch teams that rename mechanically: (1) the default model moves from gemini-1.5-pro to gemini-3-pro, so prompts tuned against 1.5 should be pinned with --model gemini-1.5-pro; (2) the --stream flag now emits SSE-formatted events instead of plain-text deltas, breaking any parser; (3) MCP server config moves out of settings.json's inline url field into a separate mcp_config.json with the field renamed to serverUrl; (4) exit codes and state directories change, so CI pipelines, cron jobs, and orchestration bridges may fail silently rather than loudly. The quota downgrade is the part that hurts heavy users most: Gemini CLI's 1,000 requests/day becomes a weekly compute-based cap that early reports say burns out in a few sessions and then enforces multi-day cooldowns. Two structural takeaways: Google is consolidating its agentic surface into one platform name, and the open-source-CLI era of Gemini is functionally over for non-enterprise users.

AI Research & Scientific Benchmarks

OpenAILifeSciBench: 750 expert-authored life-science tasks; GPT-Rosalind takes 36.1% headline pass rate, leads on 386 of 750 tasks

17 June 2026 · Primary sources: OpenAI Newsroom & OpenAI on X · Confirmed by MarkTechPost, StartupHub.ai, Cryptopond

OpenAI on Wednesday 17 June released LifeSciBench, a 750-task benchmark for evaluating how AI systems handle real-world life-science research tasks — built with 173 scientists from biotech and pharma, each with industry experience advancing drug-discovery programs, and structured across seven workflows and seven biological domains. Tasks underwent an unusually heavy review pipeline: ~6 automated reviews on average plus at least 2 rounds of expert evaluation. The companion Molecule.one announcement showed a near-autonomous AI chemist running on GPT-5.4 improving a key medicinal-chemistry reaction in a real wet-lab loop. Headline results: GPT-Rosalind (the rebrand of the formerly-internal GPT-Researcher line) raised the overall pass rate from 25.7% (GPT-5.5) to 36.1%; it had the highest per-task mean on 386 of 750 tasks. Gemini 3.1 Pro uniquely led on 214 tasks — not a sweep, but a structurally significant footprint for a non-OpenAI model on an OpenAI-published benchmark. The toughest categories were Design, Optimization & Prediction (GPT-Rosalind 30.7%) and Analysis (30.3%). Most useful diagnostic: GPT-Rosalind drops from 45.1% on text-only tasks to 28.1% on artifact tasks — meaning anything that requires actually producing a parseable scientific artifact (a SMILES string, a SLURM script, a protocol JSON) is still where these models give up the most ground. LifeSciBench is the most rigorous publicly-released life-science benchmark to date; expect it to become the citation of record for Pharma-AI vendor claims for the next year, in the same way SWE-Bench Verified became the citation of record for coding agents.

Open-Source AgentsNous Research ships async subagents in Hermes Agent — delegated work no longer blocks the parent chat

Shipped 15 June 2026 · Tier-2 coverage wave 16–17 June · Primary source: Hermes Agent GitHub Issue #5586 · Confirmed by MarkTechPost, TechTimes, FutureSignal

Nous Research shipped async subagents for Hermes Agent — the open-source autonomous agent currently leading OpenRouter's global weekly token rankings — on Monday 15 June, with a coverage wave that crested across 16–17 June. The mechanic: until now, parent-to-child delegation was sequential, so a parent agent calling a subagent froze its own conversation until the child finished. The new async_delegation toolset (tracked in GitHub Issue #5586) inverts that: the parent's tool call now returns a task_id immediately, the subagent runs as an in-process thread with its own isolated conversation, terminal session, and toolset, and only the final summary returns to the parent. The parent can keep chatting, take new instructions, or spawn more subagents in parallel. Why this matters for the agent-tools ecosystem: every credible long-running coding-agent harness — including Claude Code's subagent system documented in Pliny's leaked Fable 5 system prompt (covered in the 15 June digest) — depends on a non-blocking parent/child structure to scale beyond single-thread work. Hermes Agent is the most-deployed open-source implementation of that structure. The async-delegation toolset is now the reference design for everyone building MCP-server-driven, open-weights agent harnesses on top of the new GitHub Copilot App, Antigravity CLI, Codex CLI, or vanilla MCP runtimes.

GitHubSecret Scanning June 2026 release: Cloudsmith and Meraki added as partners, GitLab token coverage expanded, push-protection detectors for Elastic / Slack / Supabase / Datadog / VolcEngine

17 June 2026 · Primary source: GitHub Changelog · Confirmed by Releasebot, BuildMVPFast

GitHub published its June 2026 Secret Scanning update on Wednesday 17 June. Two new partner program integrations added — Cloudsmith (package-management platform) and Meraki (Cisco Meraki cloud-managed networking) — meaning any token of those types found in a public repository is now automatically reported to the issuer for revocation. GitLab token coverage was significantly expanded, closing a long-standing gap where personal access tokens, deploy tokens, and group runner tokens were not fully covered. New push-protection detectors — meaning the commit is blocked client-side before reaching the repository — were added for Elastic, Slack, Supabase, Datadog, and VolcEngine. Several of the new detectors also support validity checks, meaning alerts include whether the leaked credential is still live, which collapses the triage backlog. Push protection is on by default for repositories with secret scanning enabled, including free public repos. The pragmatic significance: with the GitHub Copilot App GA shipped the same day (see above), the surface of credentials produced and committed by autonomous agents is expanding fast, and Slack/Datadog/Supabase are exactly the surface that breaks first when an agent is misconfigured. This is GitHub deliberately tightening the loop before agentic code-commit volume scales.
