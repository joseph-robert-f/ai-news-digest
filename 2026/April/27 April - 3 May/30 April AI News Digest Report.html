Open-Weights & Model Releases

PoolsideLaguna XS.2 & M.1: Apache-2.0 agentic-coding models hit 68.2% / 72.5% on SWE-bench Verified

28 April 2026 · Primary source: Poolside

U.S. lab Poolside released its first open-weight model, Laguna XS.2, a 33B-total / 3B-active MoE trained in-house and shipped under Apache 2.0. It runs on a single GPU (and a 36 GB Mac), and posts 68.2% on SWE-bench Verified, 62.4% on SWE-bench Multilingual, 44.5% on SWE-bench Pro, and 30.1% on Terminal-Bench 2.0. A larger sibling, Laguna M.1, is API-only for now and reaches 72.5% on SWE-bench Verified. Architecturally, XS.2 mixes 3:1 sliding-window / global attention across 40 layers with sigmoid gating — a design choice that signals where local-first agentic-coding stacks are headed.

Spring AIFive new CVEs disclosed — including HIGH-severity vector-store injection

27 April 2026 (continued impact) · Primary source: VMware Tanzu / spring.io advisories

The Spring security team disclosed five Spring AI CVEs affecting versions 1.0.x and 1.1.x, with fixes shipped in 1.0.6 / 1.1.5. The two HIGH-severity issues are vector-store query-injection bugs (CVE-2026-22729 in FilterExpressionConverter, CVE-2026-40978 SQL injection in CosmosDBVectorStore) where keys/values are not properly escaped on the way to vector-store query languages. A MODERATE issue (CVE-2026-40966) lets an attacker pivot across conversationId filter logic to exfiltrate other users' chat history; two PDF-handling DoS bugs round out the batch. Anyone running RAG/agent stacks on Spring AI should treat the upgrade as this-week work.

CursorCursor 3.0 ships an "Agents Window" and Design Mode — the IDE becomes an orchestrator

In-month release; secondary coverage 28–30 April 2026 · Primary source: Cursor changelog

Cursor 3.0 reframes the editor around an Agents Window that lets developers run many agents in parallel across local repos, worktrees, the cloud, and remote SSH targets — a step beyond Cursor 2.x's eight-agent Composer model. A new Design Mode (⌘⇧D) lets you annotate UI elements directly in the browser instead of describing them in text, and Agent Tabs allow side-by-side / grid views of multiple chats. InfoQ characterises it as a deliberate move past the IDE metaphor toward agent orchestration; the traditional editor is now optional surface area, not the default.
