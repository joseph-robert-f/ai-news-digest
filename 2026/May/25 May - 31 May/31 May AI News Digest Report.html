Developer Platforms & SDKs

AnthropicDynamic Workflows in Claude Code: orchestrate up to 1,000 subagents via generated JavaScript — plan lives in code, not context

28 May 2026 (research preview) · Primary source: Claude blog / Claude Code docs · MarkTechPost / Reworked / SecurityBrief

The architectural change is the news. Dynamic Workflows ship Claude generating a JavaScript orchestration script for a user-described task and then handing that script to a sandboxed runtime that executes it in the background. The runtime supports up to 16 concurrent subagents per session and caps each run at 1,000 agents total; agents propose solutions from independent angles, refute each other's findings, and iterate until they converge. The pattern Anthropic is selling: "scripts replace context windows." Intermediate results live in script variables, not in Claude's working memory, so only the final synthesis is returned to the user-facing model — effectively unbounding context for many real engineering jobs (codebase-wide bug investigations, security reviews, large migrations, multi-file refactors). Requires Claude Code v2.1.154 or later; available in the Claude Code CLI, Desktop, and VS Code extension on Max, Team, and Enterprise plans and via the Claude API, Amazon Bedrock, Vertex AI, and Microsoft Foundry. On Max and Team plans, Dynamic Workflows are on by default. Pricing is metered by the existing per-agent token cost — the framework itself is free; the agents are not.
