OpenAICodex preview lands inside the ChatGPT mobile app on iOS and Android

15 May 2026 · Primary source: OpenAI Developers Changelog / OpenAI

OpenAI added a Codex preview directly inside the ChatGPT mobile app, letting developers continue, steer, and approve Codex coding sessions from a phone while the actual work runs on a connected Mac. From mobile, users can start or continue threads, change direction, approve actions, and review what Codex has done — with the host machine streaming live project context, diffs, screenshots, terminal output, and test results into the app. Setup is a QR-code pairing between the host Codex app and the mobile client. Rollout is on iOS and Android across all plans (including Free and Go) in supported regions, with the caveat that the Mac host has to stay awake and online for remote sessions to continue. The release is OpenAI's most direct mobile-first answer to Cursor/Claude Code on the desktop and pairs with the previously announced GPT-5.2-Codex model.

GitHubCodeQL 2.25.4: Vercel serverless coverage, Swift 6.3.1, and data-flow barriers in every language

12 May 2026 · Primary source: GitHub Changelog / CodeQL docs

GitHub shipped CodeQL 2.25.4, with first-class support for @vercel/node serverless functions — handlers are now recognized via the VercelRequest/VercelResponse TypeScript parameter types, and the core security queries (js/reflected-xss, js/request-forgery, js/sql-injection, js/command-line-injection) flow through Vercel API routes. The release also adds Swift 6.3.1 support, accuracy improvements in C#, and — arguably the biggest under-the-hood change — a data-flow barrier extension mechanism across C/C++, C#, Go, Java/Kotlin, JS/TS, Python, Ruby, and Rust, letting teams suppress false positives via data extensions without forking queries. The default suite now runs 496 queries across 169 CWEs; the extended suite adds another 131 queries / 32 CWEs.

OpenAI"Safety summaries": ChatGPT trained to surface evolving risk in sensitive long conversations

14 May 2026 · Primary source: OpenAI

OpenAI rolled out a model-side update aimed at the long, multi-turn failure mode that has driven the bulk of recent legal and regulatory pressure on ChatGPT. The release introduces safety summaries — short, factual notes about earlier safety-relevant context, generated by a model trained specifically for safety reasoning, retained only briefly, and surfaced only when a serious safety concern is detected. OpenAI reports +50% on safe-response rates in suicide / self-harm scenarios and +16% in harm-to-others scenarios on long single-conversation evaluations. The update lands as litigation over teen self-harm and emotional-dependence cases continues to mount, and as Anthropic, xAI, and Microsoft have all signed early-access agreements with the U.S. Center for AI Standards and Innovation.
