Defense, Policy & Geopolitics

DoWPentagon clears eight AI vendors for IL6/IL7 classified networks — Anthropic shut out

1 May 2026 · Primary source: U.S. Department of War press release

The U.S. Department of War announced contract awards to Amazon Web Services, Google, Microsoft, NVIDIA, OpenAI, SpaceX, and Reflection for deployment of frontier AI on networks classified at Impact Level 6 (Secret) and Impact Level 7 (Top Secret). Oracle was added to the list within hours, bringing the count to eight. Models will be served through the Department's GenAI.mil central platform and used for data synthesis, intelligence analysis, mission planning, and decision support. CDAO chief Doug Matty framed the package as a deliberate "no vendor lock-in" architecture for the Joint Force. Conspicuously absent: Anthropic, whose Claude is already running inside DoW via Palantir's Maven toolkit but which the administration designated a "supply-chain risk" earlier this year after Anthropic refused unrestricted use for autonomous weapons and domestic surveillance. Anthropic still holds a March preliminary injunction against the blacklisting; Pentagon CTO Michael Kratsios reiterated on the same day that the company remains barred regardless of separate Mythos-related discussions.

Open Research & Model Engineering

Moonshot AIFlashKDA open-sourced: 1.72–2.22× prefill speedup for Kimi Delta Attention

30 April 2026 · Primary source: MoonshotAI / FlashKDA GitHub

Moonshot AI open-sourced FlashKDA, a high-performance CUTLASS kernel implementation of Kimi Delta Attention — the linear-attention mechanism behind Kimi Linear (48 B total / 3 B active hybrid model). FlashKDA delivers 1.72×–2.22× prefill speedup over the flash-linear-attention baseline on NVIDIA H20 GPUs and supports variable-length batching, fixed head dim K=V=128, SM90+ hardware, CUDA 12.9+, and PyTorch 2.4+. Crucially, it is auto-dispatched from flash-linear-attention's chunk_kda, so projects already on FLA pick up the speedup with no code change. The release further hardens Moonshot's open-weights / open-kernel posture in the wake of last week's Kimi K2.6 long-horizon agent release.

Developer Tooling

GitHubCopilot CLI 1.0.40 ships with the long-awaited large-attachment CPU-hang fix

1 May 2026 · Primary source: GitHub Releases

GitHub Copilot CLI v1.0.40 shipped today across Windows, macOS, and Linux. The headline change for everyday users is a hard fix for the bug where the CLI would peg at 100% CPU on large file attachments; alongside it, assistant-response streaming is smoother, startup is faster (asynchronous certificate loading), and slash-command suggestions now rank prefix matches above fuzzy matches. For agent builders, the release adds client_credentials OAuth grant support for MCP servers, lifts the /research command onto an orchestrator/sub-agent model, exposes Skills as ACP slash commands, and ships a live plan display for ACP clients during multi-step tasks. Autopilot mode now defaults to a 5-message continuation cap. The release lands the same week GitHub confirmed that Copilot Code Review will start consuming GitHub Actions minutes on June 1 as part of the broader move to usage-based billing.
